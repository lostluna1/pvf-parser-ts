 - AI角色工具未单独实现
 - 使用DNF客户端程序


1. 生成物品表（包括装备和时装）
 - 为了方便使用GM命令（add item, add avatar），需要物品表
 - 物品表保存在 /Script/ItemName.lst 文件中
 - 如果客户端程序启动时没有物品表文件，会自动生成
 - 如果因物品添加或更改需要更新物品表文件，请使用GM命令（相关命令：save list item）


2. 以单人模式运行客户端时存在限制条件。


3. 保存AI文件
 - 使用GM命令 save ai 将用户角色的设定保存为AI文件（*.aic）
 - 保存的信息包括快捷栏物品、快捷栏技能、装备的装备、装备的时装
 - 保存的等级信息是原始等级（在相关因素更改之前）
 - 信息中装备的等级会被保存，但在生成AI角色时该值不会被应用，而是随机应用。

4. 相关GM命令
 - save list item : 将物品表保存为文件

 - add item [物品ID/物品名称] [物品数量（可省略）]：向用户背包中添加物品
	物品名称需用`符号标注，如`布片`
	物品数量可省略，默认值为100
	物品数量仅在以单人模式运行时适用

 - add avatar [时装ID/时装名称] [使用期限（可省略）] [附加能力选择（可省略）]：向时装衣柜中添加时装
	时装名称需用`符号标注，如`布片`
	使用期限和能力选择可省略
	使用期限为0时为无限制，1及以上为可用天数。默认值为0
	附加能力选择为-1时无附加能力，0~3为对应的附加能力。默认值为-1

 - save ai [文件名]：将用户角色的设定保存为AI文件
	文件名可省略，默认值为`TestAI`
	保存路径为 /Script/AICharacter/，文件名无需扩展名

5. 其他
 - 在 /Script/AICharacter/AICharater.lst 文件中添加AI角色文件时，
	请注意根据角色类别分配编号（参见该文件注释）
 - 生成物品表的原因：为了减少加载时间（从物品脚本文件读取物品名称需要1~2分钟，而从物品表文件读取仅需约1秒）。
